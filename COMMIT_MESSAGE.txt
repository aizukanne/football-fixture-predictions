feat(api): include actual match scores in league search results

Added the 'goals' field to the /predictions endpoint response when querying
by country and league. Previously, actual scores were only included when
querying by fixture_id.

Changes:
- Modified src/services/data_formatter.py
- Updated _format_single_fixture() to include goals field if available
- Actual scores now appear in league query responses for finished matches

Benefits:
- Consumers can now see actual results alongside predictions in bulk queries
- Enables comparison of predictions vs actual outcomes in league views
- Maintains backward compatibility (field only present if data exists)

Response format example:
{
  "fixture_id": 123,
  "home": { "predicted_goals": 1.8 },
  "away": { "predicted_goals": 1.2 },
  "goals": { "home": 2, "away": 1 }  // NEW - only if match finished
}
