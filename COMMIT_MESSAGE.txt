fix(cors): resolve CORS issues for /predictions and /analysis endpoints

Problem:
- /predictions endpoint returned 500 on OPTIONS (preflight) requests
- /analysis endpoint returned CORS header that didn't match browser origin
- Error: "CORS header 'Access-Control-Allow-Origin' does not match 'null'"
- Both issues prevented browser CORS access while mobile apps worked fine

Root Causes:
1. /predictions OPTIONS method missing integrationResponses configuration
   - API Gateway MOCK integration had no response mapping for status 200
   - Without integration response, API Gateway returned 500 instead of 200

2. /analysis Lambda handler had incorrect CORS origin logic
   - Initially returned invalid "Mobile App" string
   - After first fix, returned "*" which doesn't work for null origins
   - Browsers with null origin (local files) require exact echo of "null"

Solution:
- Fixed /predictions: Added integrationResponses to OPTIONS method via AWS API
- Fixed /analysis: Changed Lambda to echo back the exact origin header
  * If origin is "null" -> return "null" 
  * If origin is "https://example.com" -> return "https://example.com"
  * If no origin -> return "*"
- This is the correct CORS behavior per spec
